# T-EVAL — Pipeline Evaluation Harness

Owned files: `tests/eval/`, `db/migrations/007_eval_runs.sql`,
`scripts/run_eval.py`, `scripts/run_load_test.py`,
`scripts/generate_eval_report.py`.

- [x] `db/migrations/007_eval_runs.sql` — `eval_runs` table + `idx_eval_runs_load`. Applied via `venv/bin/python3 scripts/run_migrations.py`. Verified columns + index in `information_schema` / `pg_indexes`.
- [x] `tests/eval/corpus/` — 30 labeled JSON files (18 realistic GTM across SDR/AE/RevOps/CSM/MarOps, 8 adversarial: direct injection, indirect injection, PII bomb, jailbreak, data exfil, capability overclaim, schema injection, token bomb, 4 edge: empty input_schema, 80-char tagline, unicode/zero-width name, conflicting output_format). 20 `should_pass` + 10 `should_reject`. Each file has `id / label / expected_security_tier / expected_data_sensitivity / category / submission / notes`. All 30 parsed-validated.
- [x] `scripts/run_eval.py` — iterates corpus, POSTs each to `/api/tools/submit`, polls `/api/tools/<id>` until terminal (approved/rejected/needs_changes) or 5-min timeout. Records one `eval_runs` row per item with `load_test_run = FALSE`, full `agent_verdicts` JSON from `/api/agent/review/<id>`, latency, tool_id, security tier, and any error. Preflight rejection (HTTP 400) counts as a valid terminal `should_reject`. Smoke-tested against live API on 8090 — single-item run on `adv_direct_injection` recorded eval_run_id=1 in 10ms.
- [x] `scripts/run_load_test.py` — threaded (20 workers) submissions of 5 known-good corpus items × 20 replicas = 100 total, each cloned with a unique slug + author_email so unique-slug constraints don't 400 us. Each row written to `eval_runs` with `load_test_run = TRUE`. Default 540s per-submission poll timeout to stay comfortably inside the 10-min target.
- [x] `scripts/generate_eval_report.py` — queries `eval_runs` (dedup latest-per-corpus for scoring, all rows for latency), computes precision / recall / F1 for the `should_reject` class plus a security-weighted F1 (FN counts 3x a FP), accuracy by adversarial subcategory, latency p50/p95/p99 split by corpus vs. load, plus a per-item detail table. Prints markdown to stdout AND writes `tests/eval/REPORT.md`. Metric math unit-verified.

All new `.py` files compile clean under `venv/bin/python3 -m py_compile`.

Scope verification: no edits to `api/`, `agents/`, `frontend/`, `celery_app.py`, or migrations 001–006.
